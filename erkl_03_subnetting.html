Netzwerkpräfix & Subnetting

Erklärung · Schritt für Schritt zum Verständnis

Abschnitt 1

Aufbau einer IPv6-Adresse

Eine IPv6-Adresse ist 128 Bits lang. Sie besteht aus genau zwei Teilen:

  • Netzwerkpräfix (Network Prefix) — identifiziert das Netzwerk, in dem sich das Gerät befindet.
  • Interface Identifier (IID) — identifiziert das einzelne Gerät (Host) innerhalb dieses Netzwerks.

Wo genau die Grenze zwischen Präfix und IID liegt, bestimmt die Präfixlänge. Sie wird als Zahl nach einem Schrägstrich geschrieben, z. B. /64.

2001:0db8:abcd:0012:0000:0000:0000:0001 /64
Netzwerkpräfix (64 Bits) Interface ID (64 Bits)
Präfix · 64 Bits
Interface ID · 64 Bits
/64 ist der Standardschnitt für Endnutzer-Netzwerke (LANs). Die ersten 64 Bits gehören zum Netzwerk, die letzten 64 Bits identifizieren den Host.
Abschnitt 2

Präfixlänge verstehen

Die Präfixlänge gibt an, wie viele Bits von links gezählt zum Netzwerkpräfix gehören:

  • /48 = die ersten 48 Bits sind der Netzwerkpräfix
  • /64 = die ersten 64 Bits sind der Netzwerkpräfix
  • /128 = alle 128 Bits sind fixiert → eine einzige Adresse

Faustregel: Je größer die Zahl nach dem /, desto spezifischer das Netzwerk — weniger Hosts, aber mehr mögliche Subnetze.

Analogie: Stell dir eine Postleitzahl vor. „0“ ist ganz Deutschland, „04“ ist Leipzig und Umgebung, „04103“ ist ein einzelner Stadtteil. Je mehr Ziffern, desto genauer der Bereich — genauso funktioniert die Präfixlänge.

Abschnitt 3

Subnetting — Subnetze berechnen

Subnetting bedeutet: Einen großen Adressblock in kleinere Subnetze aufteilen. Die zentrale Formel:

Anzahl Subnetze = 2(Zielpräfix − Ausgangspräfix)
z. B. /48 → /64: 2^(64 − 48) = 2^16 = 65.536 Subnetze

Die Bits zwischen dem Ausgangspräfix und dem Zielpräfix (/64) sind die „Subnet-Bits“. Jedes dieser Subnetze erhält seinen eigenen 64-Bit-Interface-ID-Bereich.

Beispiel: /48 → /64 Subnetze

Präfix · 48
Subnet · 16
IID · 64
Netzwerkpräfix (48 Bit) Subnet-Bits (16 Bit) Interface ID (64 Bit)

Schritt-für-Schritt-Beispiel

Gegeben: 2001:db8:cafe::/48
1
Das Präfix ist 48 Bits lang. Für /64-Subnetze bleiben:
64 − 48 = 16 Subnet-Bits
2
Anzahl Subnetze berechnen:
2^16 = 65.536 mögliche /64-Subnetze
3
Die 16 Subnet-Bits bilden das 4. Hextet (Positionen 49–64). Indem wir dieses Hextet von 0000 bis ffff durchzählen, erhalten wir alle Subnetze:
1. Subnetz: 2001:db8:cafe:0000::/64
2. Subnetz: 2001:db8:cafe:0001::/64
3. Subnetz: 2001:db8:cafe:0002::/64
···
Letztes Subnetz: 2001:db8:cafe:ffff::/64
Abschnitt 4

Subnetting mit anderen Präfixlängen

Das Prinzip funktioniert auch mit anderen Ausgangspräfixen. Die Subnet-Bits erstrecken sich dann über mehrere Hextets.

/44 → /64 Subnetze

2(64 − 44) = 220 = 1.048.576 Subnetze
Präfix · 44
Subnet · 20
IID · 64

Die 20 Subnet-Bits erstrecken sich über die letzten 4 Bits des 3. Hextets und die gesamten 16 Bits des 4. Hextets.

Beispiel: 2001:db8:a000::/44
1. Subnetz: 2001:db8:a000:0000::/64
2. Subnetz: 2001:db8:a000:0001::/64
3. Subnetz: 2001:db8:a000:0002::/64
···
Letztes Subnetz: 2001:db8:afff:ffff::/64

/36 → /64 Subnetze

2(64 − 36) = 228 = 268.435.456 Subnetze
Präfix · 36
Subnet · 28
IID · 64

Die 28 Subnet-Bits erstrecken sich über die letzten 12 Bits des 3. Hextets und die gesamten 16 Bits des 4. Hextets.

Beispiel: 2001:db8:b000::/36
1. Subnetz: 2001:db8:b000:0000::/64
2. Subnetz: 2001:db8:b000:0001::/64
3. Subnetz: 2001:db8:b000:0002::/64
···
Letztes Subnetz: 2001:db8:bfff:ffff::/64
Abschnitt 5

Hosts pro Subnetz

In einem /64-Subnetz stehen 64 Bits für die Interface-ID zur Verfügung:

Hosts = 2(128 − Präfixlänge)
z. B. /64: 2^(128 − 64) = 2^64 = 18.446.744.073.709.551.616 Adressen

Das sind über 18 Trillionen mögliche Adressen pro Subnetz — eine gigantische Zahl!

Wichtiger Unterschied zu IPv4: Bei IPv6 werden keine Netzwerk- oder Broadcast-Adressen abgezogen. Alle Adressen im IID-Bereich können theoretisch verwendet werden.

Abschnitt 6

Warum /64 als Standard?

Es gibt drei zentrale Gründe, warum /64 in der IPv6-Welt als Standardpräfix für Endnutzer-Subnetze gilt:

  1. SLAAC benötigt 64 Bits für die Interface-ID
    Stateless Address Autoconfiguration (SLAAC) erzeugt die Interface-ID automatisch — entweder über EUI-64 (aus der MAC-Adresse) oder über Privacy Extensions (zufällig). Beide Verfahren erwarten genau 64 Bits.
  2. Standardisierung vereinfacht die Netzwerkplanung
    Wenn jedes Subnetz /64 ist, entfällt die Frage nach der „richtigen“ Subnetzgröße. Das macht Planung, Dokumentation und Troubleshooting einfacher.
  3. Viele IPv6-Funktionen setzen /64 voraus
    Neighbor Discovery, SLAAC und andere Protokolle sind für /64-Subnetze entworfen. Andere Präfixlängen können zu Kompatibilitätsproblemen führen.
Abschnitt 7

Tipps für die Klausur

1. Adressen immer vollständig ausschreiben
Beim Rechnen keine abgekürzten Adressen verwenden. Erst expandieren, dann die Bits zählen.
2. Bits sorgfältig zählen
Präfix-Bits + Subnet-Bits + Host-Bits = 128
Diese Gleichung muss immer aufgehen!
3. Subnetzadressen: Host-Teil ist Null
Die Netzwerkadresse eines Subnetzes hat immer alle Host-Bits (IID) auf 0 gesetzt.
4. Formel einprägen
Subnetze = 2(Zielpräfix − Ausgangspräfix)
Hosts = 2(128 − Präfixlänge)
