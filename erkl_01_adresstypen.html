Adresstypen & Präfixe

Erklärung · Das Wichtigste zu den IPv6-Adresstypen

Grundlagen

Was ist eine IPv6-Adresse?

Die Basis, bevor wir über Typen sprechen.

Eine IPv6-Adresse ist 128 Bit lang – viermal so viel wie eine IPv4-Adresse mit ihren 32 Bit. Das ergibt einen unvorstellbar grossen Adressraum von ca. 3,4 × 1038 Adressen.

  • 8 Gruppen (sogenannte Hextets) zu je 16 Bit
  • Getrennt durch Doppelpunkte :
  • Darstellung in Hexadezimalzahlen (Ziffern 0–9 und Buchstaben a–f)
  • Jedes Hextet besteht aus 4 Hex-Zeichen (= 16 Bit)
2001: 0db8: 85a3: 0000: 0000: 8a2e: 0370: 7334
Netzwerk-Präfix (64 Bit)
Interface ID (64 Bit)
Netzwerkteil Hostteil (Interface ID)
Merkhilfe
IPv4 = 4 Dezimalzahlen mit Punkten: 192.168.1.1
IPv6 = 8 Hex-Gruppen mit Doppelpunkten: 2001:0db8:…:7334
Die 5 Adresstypen

Welche Typen gibt es?

Jede IPv6-Adresse gehört zu einem bestimmten Typ – erkennbar am Präfix (den ersten Zeichen).

1. Global Unicast

Global Unicast 2000::/3
Vergleich: Wie eine öffentliche IPv4-Adresse – weltweit eindeutig und routbar.

Global-Unicast-Adressen sind die „normalen“ Internet-Adressen. Jedes Gerät, das direkt im Internet erreichbar sein soll, bekommt eine solche Adresse.

  • Weltweit routbar – funktioniert überall im Internet
  • Das Präfix 2000::/3 bedeutet: alle Adressen, die mit 2 oder 3 beginnen (binär: 001…)
  • Aktuell sind fast alle vergebenen Adressen im Bereich 2xxx:…
  • Wird vom Provider (ISP) oder einer Registry (RIPE, ARIN, etc.) zugeteilt
Beispiele 2607:f8b0:4004:0800::200e ← Google DNS
2a00:1450:4001:081f::2004 ← Google Server

2. Link-Local

Link-Local fe80::/10
Vergleich: Wie eine APIPA-Adresse (169.254.x.x) bei IPv4 – automatisch, nur lokal.

Link-Local-Adressen werden automatisch auf jedem IPv6-fähigen Interface konfiguriert – ohne DHCP, ohne manuelle Einstellung. Sie funktionieren nur im selben Netzwerksegment (Link).

  • Nicht routbar – Pakete verlassen nie das lokale Segment
  • Beginnt immer mit fe80::
  • Wird verwendet für: Neighbor Discovery, Router Discovery, Grundkommunikation im LAN
  • Jedes IPv6-Interface hat immer eine Link-Local-Adresse, auch ohne Router
Beispiele fe80::1 ← typisch für Router-Interface
fe80::a1b2:c3d4:e5f6:7890 ← per EUI-64 generiert

3. Unique Local (ULA)

Unique Local fc00::/7
Vergleich: Wie 192.168.x.x oder 10.x.x.x bei IPv4 – private Adressen für interne Netze.

Unique-Local-Adressen sind für private Netzwerke gedacht. Sie werden im Internet nicht geroutet und sind nur innerhalb der eigenen Organisation gültig.

  • Nicht im Internet routbar
  • Offizielles Präfix: fc00::/7 – in der Praxis wird fast immer fd00::/8 verwendet
  • Ideal für: Firmennetze, Labore, Heimnetzwerke
  • Der 40-Bit-Teil nach fd sollte zufällig gewählt werden, um Konflikte bei Zusammenschaltung zu vermeiden
Beispiele fd12:3456:789a::1 ← typische ULA-Adresse
fd00::1 ← einfache ULA

4. Multicast

Multicast ff00::/8
Vergleich: Wie Broadcast bei IPv4 – aber gezielter. Eine Nachricht an eine Gruppe statt an alle.

Multicast sendet ein Paket gleichzeitig an mehrere Empfänger, die zu einer Gruppe gehören. In IPv6 gibt es keinen Broadcast mehr – Multicast übernimmt diese Rolle.

  • Beginnt immer mit ff
  • Das dritte Hex-Zeichen definiert den Scope (Reichweite):
ff01::Node-Local (nur das eigene Gerät)
ff02::Link-Local (lokales Segment)
ff05::Site-Local (ganzer Standort)
ff0e::Global (gesamtes Internet)
Wichtige Multicast-Adressen
ff02::1 – Alle Nodes im lokalen Segment
ff02::2 – Alle Router im lokalen Segment
Beispiele ff02::1 ← All Nodes (Link-Local)
ff02::2 ← All Routers (Link-Local)
ff02::1:ff00:1 ← Solicited-Node Multicast

5. Dokumentation / Beispiel

Dokumentation 2001:db8::/32
Vergleich: Wie 192.0.2.0/24 bei IPv4 – nur für Beispiele in Dokumentationen gedacht.

Dieser Adressbereich ist ausschliesslich für Dokumentation und Beispiele reserviert. Er wird niemals in echten Netzwerken verwendet und ist nicht routbar.

  • Erkennbar an 2001:db8: bzw. 2001:0db8: am Anfang
  • Wird in RFCs, Lehrbüchern, Prüfungen und Tutorials verwendet
  • Wenn du in einer Klausur eine Adresse mit 2001:db8: siehst – das ist Dokumentation!
Beispiele 2001:0db8:85a3::8a2e:0370:7334 ← klassisches Doku-Beispiel
2001:db8:1234::1 ← vereinfacht
2001:db8::/32 ← der gesamte reservierte Bereich
Bonus

Anycast

Ein spezieller Adresstyp ohne eigenes Präfix.

Anycast ist besonders: Es gibt kein eigenes Präfix. Anycast-Adressen stammen aus dem normalen Unicast-Adressraum. Das Besondere ist die Verwendung, nicht die Adresse selbst.

  • Mehrere Hosts teilen sich dieselbe Adresse
  • Ein Paket wird immer zum nächstgelegenen Host geroutet (basierend auf Routing-Metrik)
  • Das Netzwerk entscheidet automatisch, welcher Host antwortet

Typische Einsatzgebiete:

  • DNS-Server – z.B. nutzt Cloudflare die Anycast-Adresse 1.1.1.1
  • CDN (Content Delivery Networks) – Inhalte werden vom nächsten Server geliefert
  • Lastverteilung – Anfragen verteilen sich automatisch auf mehrere Standorte
Klausur-Hinweis
Man kann Anycast nicht am Präfix erkennen. Es ist ein Konfigurationsmerkmal, kein Adressmerkmal. In Aufgaben wird Anycast oft als „Sonderfall“ abgefragt.

Alle Adresstypen mit Präfix, Geltungsbereich und Zweck.

Global Unicast 2000::/3 Global Öffentliche Internet-Adressen

Klausurvorbereitung

Tipps für die Klausur

So erkennst du den Adresstyp auf einen Blick.

Schnell-Erkennung: Schau dir die ersten Zeichen der Adresse an – das Präfix verrät den Typ.

  • ff… → Multicast
  • fe80:: → Link-Local
  • fd… / fc… → Unique Local
  • 2001:db8: → Dokumentation
  • 2… / 3… → Global Unicast
Achtung: Reihenfolge beachten!
2001:db8:… beginnt zwar mit 2, ist aber kein Global Unicast, sondern Dokumentation. Prüfe daher immer zuerst auf 2001:db8:, bevor du „Global Unicast“ wählst. Je spezifischer das Präfix, desto höher die Priorität.

Eselsbrücken:

  • ff = Für Freunde (Multicast = Gruppe)
  • fe80 = Festgeklebt am Link (Link-Local)
  • fd = Für Daheim (Unique Local = privat)
  • 2001:db8 = Doku-Beispiel (Dokumentation)
