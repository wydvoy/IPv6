Komprimierung & Expandierung

Erklärung · Die zwei Regeln zur IPv6-Kurzschreibweise (RFC 5952)

Grundlagen

Warum Komprimierung?

IPv6-Adressen sind lang — Komprimierung macht sie handhabbar.

Eine vollständige IPv6-Adresse besteht aus 128 Bit, dargestellt als 32 Hexadezimalzeichen in 8 Gruppen (Hextets) à 4 Zeichen, getrennt durch Doppelpunkte:

2001:0db8:0000:0000:0000:0000:0000:0001

Das sind 39 Zeichen — viel zu lang für den Alltag. Deshalb definiert RFC 5952 zwei Regeln, die Adressen deutlich kürzer machen, ohne dass Information verloren geht:

2001:db8::1

Aus 39 Zeichen werden 11 — dieselbe Adresse!

Regel 1

Führende Nullen entfernen

In jedem Hextet dürfen führende Nullen weggelassen werden.

Betrachte jedes der 8 Hextets einzeln. Alle Nullen am Anfang (links) werden entfernt. Es muss aber mindestens eine Ziffer übrig bleiben.

0042
↓
42
0db8
↓
db8
0000
↓
0
2001
↓
2001
0001
↓
1
0200
↓
200
Wichtig: 0000 wird zu 0 — nicht zu einem leeren String! Es muss immer mindestens eine Ziffer pro Hextet stehen bleiben.

Schritt-für-Schritt-Beispiel:

Vorher 0db8 : 0000 : 0000 : 0000 : 0204 : 61ff : fe9d : f156
Nachher db8 : 0 : 0 : 0 : 204 : 61ff : fe9d : f156

Vollständig: fe80:0000:0000:0000:0204:61ff:fe9d:f156 → fe80:0:0:0:204:61ff:fe9d:f156

Regel 2

Längste Null-Folge durch :: ersetzen

Eine zusammenhängende Kette von Null-Hextets kann durch :: abgekürzt werden.

Nachdem Regel 1 angewandt wurde, sucht man die längste zusammenhängende Folge von Hextets, die nur aus 0 bestehen. Diese wird durch :: (Doppel-Doppelpunkt) ersetzt.

Kritische Regel: :: darf nur EINMAL pro Adresse verwendet werden! Sonst wäre die Adresse nicht mehr eindeutig expandierbar.
Bei gleicher Länge: Wenn es zwei gleich lange Null-Folgen gibt, wird die erste (weiter links stehende) ersetzt.

Beispiele:

:: am Ende (6 Null-Hextets)
2001:db8:0:0:0:0:0:0
↓
2001:db8::
:: am Anfang (7 Null-Hextets)
0:0:0:0:0:0:0:1
↓
::1
:: in der Mitte (4 Null-Hextets)
fe80:0:0:0:204:61ff:fe9d:f156
↓
fe80::204:61ff:fe9d:f156
Zwei Null-Folgen — die erste (längste/gleich lange) wird ersetzt
2001:db8:0:0:0:ff00:0:0
↓
2001:db8::ff00:0:0
Die erste Folge (3 Hextets) ist länger als die zweite (2 Hextets) → erste wird ersetzt.
Walkthrough

Komprimierung Schritt für Schritt

Drei vollständige Beispiele von der Langform zur Kürzestform.

Beispiel 1
Vollständige Adresse
2001:0db8:0000:0000:0000:0000:0000:0001
Schritt 1 — Führende Nullen entfernen (Regel 1)
2001:db8:0:0:0:0:0:1

Eine Folge von 5 aufeinanderfolgenden Null-Hextets (Position 3–7)

Vollständige Adresse
fe80:0000:0000:0000:0204:61ff:fe9d:f156

fe80:0:0:0:204:61ff:fe9d:f156
0000→0 (3×), 0204→204

fe80:0:0:0:204:61ff:fe9d:f156

Schritt 3 — Längste Folge durch :: ersetzen (Regel 2)
fe80:::204:61ff:fe9d:f156
Ergebnis
fe80::204:61ff:fe9d:f156
Beispiel 3 — Zwei Null-Folgen
Vollständige Adresse
2001:0db8:0000:0001:0000:0000:0000:0001
Schritt 1 — Führende Nullen entfernen (Regel 1)
2001:db8:0:1:0:0:0:1
0db8→db8, 0000→0, 0001→1, 0000→0 (3×), 0001→1
Schritt 2 — Null-Folgen identifizieren
2001:db8:0:1:0:0:0:1
Folge A: 1 Null-Hextet (Position 3) — Folge B: 3 Null-Hextets (Position 5–7) → Folge B ist länger!
Schritt 3 — Längste Folge durch :: ersetzen (Regel 2)
2001:db8:0:1::1
Das einzelne 0-Hextet (Folge A) bleibt stehen — :: darf nur einmal verwendet werden!
Ergebnis
2001:db8:0:1::1
Rückweg

Expandierung — Von kurz zu lang

Die Umkehrung: Aus der Kurzform die vollständige Adresse rekonstruieren.

1

:: lokalisieren und fehlende Hextets berechnen

Eine vollständige IPv6-Adresse hat immer 8 Hextets. Zähle die vorhandenen Hextets (links und rechts von ::) und berechne:

Fehlende Hextets = 8 − vorhandene Hextets
2

:: durch die korrekte Anzahl von 0000-Hextets ersetzen

Füge an der Stelle von :: so viele 0000-Hextets ein, bis insgesamt 8 Gruppen vorhanden sind.

3

Jedes Hextet auf 4 Stellen mit führenden Nullen auffüllen

Jedes Hextet muss exakt 4 Hexadezimalziffern haben. Kürzere Hextets werden links mit Nullen aufgefüllt.

Vollständiges Beispiel:

Expandierung von 2001:db8::1
Ausgangspunkt
2001:db8::1
Schritt 1 — :: lokalisieren, Hextets zählen
Links von :: → 2001, db8 (2 Stück)
Rechts von :: → 1 (1 Stück)
Vorhanden: 2 + 1 = 3 Hextets → Fehlend: 8 − 3 = 5 Null-Hextets
Schritt 2 — :: durch 5× 0000 ersetzen
2001:db8:0000:0000:0000:0000:0000:1
Schritt 3 — Alle Hextets auf 4 Stellen auffüllen
2001:0db8:0000:0000:0000:0000:0000:0001
db8→0db8, 1→0001
Ergebnis
2001:0db8:0000:0000:0000:0000:0000:0001
Expandierung von fe80::204:61ff:fe9d:f156
Ausgangspunkt
fe80::204:61ff:fe9d:f156
Schritt 1 — :: lokalisieren, Hextets zählen
Links: fe80 (1) — Rechts: 204, 61ff, fe9d, f156 (4)
Vorhanden: 1 + 4 = 5 → Fehlend: 8 − 5 = 3 Null-Hextets
Schritt 2 — :: durch 3× 0000 ersetzen
fe80:0000:0000:0000:204:61ff:fe9d:f156
Schritt 3 — Alle Hextets auf 4 Stellen auffüllen
fe80:0000:0000:0000:0204:61ff:fe9d:f156
204→0204
Ergebnis
fe80:0000:0000:0000:0204:61ff:fe9d:f156
Achtung

Häufige Fehler

Diese Fehler kosten in der Klausur Punkte — vermeide sie!

×
:: zweimal verwenden
2001::db8::1 — Nicht erlaubt! Man kann nicht wissen, wie viele Null-Hextets an welcher Stelle fehlen.
2001:db8::1 — Korrekt.
×
Hextet komplett entfernen statt auf 0 kürzen
0000 wird zu 0 — nicht zu nichts. Pro Hextet muss mindestens eine Ziffer stehen bleiben (außer es wird durch :: ersetzt).
×
Hextets beim Expandieren falsch zählen
2001:db8::1 hat 3 sichtbare Hextets, nicht 2! Also fehlen 8 − 3 = 5 Null-Hextets. Tipp: Zähle die Doppelpunkte sorgfältig.
×
Nicht die längste Null-Folge für :: wählen
Bei 2001:0:0:1:0:0:0:1 muss die längere Folge (3 Nullen, Position 5–7) gewählt werden:
2001::1:0:0:0:1 — Falsch (2er-Folge ersetzt)
2001:0:0:1::1 — Richtig (3er-Folge ersetzt)
×
Bei gleich langen Folgen die falsche wählen
Wenn zwei Folgen gleich lang sind, wird die erste (linke) ersetzt:
2001:0:0:ff:0:0:1:1 → 2001::ff:0:0:1:1 (nicht 2001:0:0:ff::1:1)
Prüfungstipps

Tipps für die Klausur

Damit in der Prüfung nichts schiefgeht.

Formel zum Zählen: Vorhandene Hextets links und rechts von :: zählen. Fehlende = 8 − vorhandene. So weißt du genau, wie viele Null-Hextets eingesetzt werden müssen.
Übe beide Richtungen: Komprimieren und Expandieren sind gleich wichtig. In der Klausur werden beide Aufgabentypen vorkommen. Wer nur eine Richtung beherrscht, verschenkt Punkte.
Kontrollschritt: Nach dem Komprimieren: Expandiere dein Ergebnis im Kopf zurück und prüfe, ob dieselbe Langform herauskommt. Wenn ja, ist alles richtig.
Systematisch vorgehen: Immer erst Regel 1, dann Regel 2. Nicht beides gleichzeitig machen — das führt zu Fehlern.
